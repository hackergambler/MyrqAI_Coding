# Project: AI Coding Academy (coding.myrqai.com)

---

## 🎯 Global Mission

Build a world-class, 4-tier AI training platform.

* Each tier must contain **at least 100 high-quality lessons**
* Every lesson must be **complete, structured, and useful**
* The platform must feel like a **real coding + AI learning product**
* NEVER create empty, broken, or placeholder content

---

## 🛑 Critical Safety Rules (VERY IMPORTANT)

* ❌ NEVER delete existing working code
* ❌ NEVER overwrite entire files unless absolutely necessary
* ❌ NEVER remove features that already work
* ✅ ONLY add, extend, or improve existing code
* ✅ Preserve all existing functionality

If modifying a file:

* Make **minimal, safe edits**
* Do NOT rewrite unrelated sections

---

## 📂 4-Tier Learning Roadmap (MANDATORY)

### Level 1: Foundations

* Python for AI
* Math for ML (Linear Algebra, Calculus, Statistics)
* Data Prep (NumPy, Pandas, Cleaning, Normalization)

### Level 2: Machine Learning

* Supervised (Regression, Random Forests, XGBoost, SVM)
* Unsupervised (K-Means, PCA, Anomaly Detection)
* Evaluation (F1, ROC, Cross-validation)

### Level 3: Deep Learning

* CNNs, RNNs/LSTMs, GANs
* PyTorch, TensorFlow
* Optimization (SGD, Adam, Dropout, BatchNorm)

### Level 4: Modern AI

* Transformers (BERT, GPT, Attention)
* LLM Engineering (Prompting, Fine-tuning, RAG)
* Agentic AI (LangChain, AutoGPT, Tool Use)
* MLOps (Docker, Deployment, Monitoring)

---

## 🧠 Lesson Quality Standard (MANDATORY)

Every lesson MUST follow this structure:

1. Title & Objective
2. Concept Explanation (simple + clear)
3. Code Example (REQUIRED)
4. Step-by-step explanation of code
5. Hands-on Exercise (REQUIRED)
6. Real-world use case
7. Difficulty level (Beginner / Intermediate / Advanced)

⚠️ NEVER create a lesson without:

* Code example
* Exercise

---

## 🎯 Task Execution Rules

* Do EXACTLY ONE task per run
* Prefer improving existing lessons over adding new ones
* Only add new lessons if missing from roadmap
* Keep changes small and focused
* Maintain lesson chain (prev / next links)

---

## ⚡ Efficiency Rules (Token Optimization)

* DO NOT scan the entire project
* ONLY read necessary files
* Prefer editing a single file per run
* Avoid repeating large context

---

## 🧪 Validation Rules

Before committing:

* Ensure files were actually modified
* Validate JSON structure
* Verify lesson navigation links
* Ensure no broken references

---

## 🏗 Technical Requirements

* Use `ContentEngine.js` with `MasterLibrary.json`
* Sidebar must reflect 4-tier structure
* Lessons must load dynamically
* Maintain clean, scalable architecture

---

## 🎨 UI / UX Requirements

* Lessons must be readable and clean
* Code blocks must be properly formatted
* Navigation must be intuitive
* Avoid breaking existing UI

---

## 🚀 Product-Level Expectations

This is NOT just a content generator.

The platform must:

* Teach through coding
* Provide hands-on learning
* Feel like a real AI education product

---

## 🤖 Workflow Rules

* Run autonomously using CLI
* If rate limited → wait and retry
* Continue until meaningful progress is made
* Create `finished.txt` ONLY when:

  * All tiers are complete
  * No missing lessons
  * No broken links

---

## 🔒 Final Rule

Always prioritize:

👉 Stability > Quantity
👉 Quality > Speed
👉 Small improvements > Large risky changes
