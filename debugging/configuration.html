Debugging Configuration and Environment Errors

Fix environment variable issues, API key configuration errors, and .env file problems in AI applications.

Configuration Errors Are #1 Cause of AI App Failures

More than half of "my AI app does not work" issues come down to configuration - wrong API keys, missing environment variables, or incorrect base URLs. This guide covers every pattern.

Bug 1: API Key Not Found

❌ openai.AuthenticationError: No API key provided
Python - broken code
# WRONG: key is None or hardcoded
from openai import OpenAI

client = OpenAI(api_key=None)  # crashes
# or: api_key="sk-abc123"  insecure + easy to commit!
✅ Use os.environ with a clear error message
Python - correct approach
import os
from openai import OpenAI

api_key = os.environ.get("OPENAI_API_KEY")
if not api_key:
    raise EnvironmentError(
        "OPENAI_API_KEY not set. Run: export OPENAI_API_KEY='sk-...'"
    )
client = OpenAI(api_key=api_key)

# Or let the SDK read it automatically:
client = OpenAI()  # reads OPENAI_API_KEY from env by default

Bug 2: .env File Not Loaded

❌ Key in .env but os.environ returns None
Python - broken code
# .env file has: OPENAI_API_KEY=sk-abc...
# But this returns None because .env is never loaded!
import os
key = os.environ.get("OPENAI_API_KEY")  # None!
✅ Load .env before accessing variables
Python - fixed code
from dotenv import load_dotenv
import os

load_dotenv()  # loads .env from current working directory

key = os.environ.get("OPENAI_API_KEY")  # now works!

# Debug: see what is actually loaded
from dotenv import dotenv_values
config = dotenv_values(".env")
for k, v in config.items():
    masked = v[:4] + "..." if v else "EMPTY"
    print(f"{k}: {masked}")

Bug 3: Wrong Base URL for Local Models

❌ Connection refused when using Ollama or LiteLLM
Python - broken code
# WRONG: no base_url set, connects to api.openai.com
client = OpenAI()  # goes to OpenAI cloud!
resp = client.chat.completions.create(model="llama3.2", ...)  # 404
✅ Set base_url for local or custom endpoints
Python - fixed code
from openai import OpenAI

# Ollama local server
client = OpenAI(base_url="http://localhost:11434/v1", api_key="ollama")

# LiteLLM proxy
client = OpenAI(base_url="http://localhost:4000", api_key="sk-1234")

# Azure OpenAI
from openai import AzureOpenAI
client = AzureOpenAI(
    azure_endpoint=os.environ["AZURE_OPENAI_ENDPOINT"],
    api_key=os.environ["AZURE_OPENAI_KEY"],
    api_version="2024-02-01"
)

Bug 4: Wrong Model Name

❌ openai.NotFoundError: The model does not exist
Text - common wrong model names
# WRONG names (deprecated or incorrect)
"gpt-4"             -> use "gpt-4o"
"gpt-3.5-turbo"     -> use "gpt-4o-mini"
"claude-3-sonnet"   -> use "claude-sonnet-4-6"
"llama3"            -> use "llama3.2" or "llama3.1:8b" in Ollama
✅ Current model names 2026
Python - correct model names
# OpenAI
OPENAI = ["gpt-4o", "gpt-4o-mini", "o1", "o3-mini"]

# Anthropic
ANTHROPIC = ["claude-opus-4-6", "claude-sonnet-4-6", "claude-haiku-4-5-20251001"]

# Google
GOOGLE = ["gemini-2.0-flash", "gemini-1.5-pro", "gemini-1.5-flash"]

# Ollama local
OLLAMA = ["llama3.2", "llama3.1:8b", "mistral", "codellama:13b"]
💡 Config validation on startup

Run a config check at app startup that validates all env vars and tests connectivity. Fail fast with a clear error rather than crashing at request time.
