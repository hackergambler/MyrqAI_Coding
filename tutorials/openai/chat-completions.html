Chat Completions Deep Dive

The Chat Completions API is the core of OpenAI's GPT models. Learn how messages work, understand the three roles, and build multi-turn conversations.

📑 In This Lesson
  • 1. Understanding the Messages Array
  • 2. The Three Roles Explained
  • 3. Crafting Effective System Prompts
  • 4. Building Conversation History
  • 5. Key Parameters
  • 6. Practice Exercise

1. Understanding the Messages Array

Every Chat Completions request contains a messages array. This array represents the conversation history and tells GPT the context of your request:

messages = [
    {"role": "system", "content": "You are a helpful coding assistant."},
    {"role": "user", "content": "How do I read a file in Python?"},
    {"role": "assistant", "content": "Here's how to read a file..."},
    {"role": "user", "content": "Can you show me with error handling?"}
]

The API processes this entire conversation to understand context and generate an appropriate response.

2. The Three Roles Explained

🎯 System Role

Sets the behavior, personality, and constraints for the AI. Think of it as the "configuration" for the assistant. Only include one system message at the start.

👤 User Role

Represents messages from the human user. These are the questions, requests, or inputs that the AI should respond to.

🤖 Assistant Role

Represents previous AI responses. Include these to give the model context about what it has already said in the conversation.

3. Crafting Effective System Prompts

The system prompt is incredibly powerful. Here are examples for different use cases:

# For a coding assistant:
system_prompt = """You are an expert Python developer. 
- Always provide working, tested code
- Include comments explaining the code
- Mention potential edge cases
- Suggest improvements when relevant"""

# For a customer service bot:
system_prompt = """You are a friendly customer service representative for TechCorp.
- Be helpful and empathetic
- If you don't know something, say so
- For billing issues, direct users to billing@techcorp.com
- Keep responses concise but complete"""

# For a creative writing assistant:
system_prompt = """You are a creative writing coach.
- Help users develop their stories and characters
- Give constructive feedback
- Suggest techniques from famous authors
- Encourage experimentation"""

4. Building Conversation History

For multi-turn conversations, you need to maintain and send the full history:

from openai import OpenAI

client = OpenAI()

# Initialize conversation with system prompt
conversation = [
    {"role": "system", "content": "You are a helpful assistant."}
]

def chat(user_message):
    # Add user message to history
    conversation.append({"role": "user", "content": user_message})
    
    # Get response
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=conversation
    )
    
    assistant_message = response.choices[0].message.content
    
    # Add assistant response to history
    conversation.append({"role": "assistant", "content": assistant_message})
    
    return assistant_message

# Example conversation
print(chat("My name is Alex"))
print(chat("What's my name?"))  # It remembers!
💡 Managing Long Conversations

As conversations grow, you'll hit token limits. Strategies include: summarizing older messages, using a sliding window of recent messages, or storing conversation in a database and retrieving relevant parts.

5. Key Parameters

Beyond messages, these parameters control the output:

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=messages,
    
    # Temperature: 0-2, controls randomness
    # 0 = deterministic, 1 = balanced, 2 = very random
    temperature=0.7,
    
    # Max tokens: limit response length
    max_tokens=500,
    
    # Top P: alternative to temperature (use one or the other)
    # 0.1 = consider top 10% probable tokens
    top_p=1.0,
    
    # Stop sequences: stop generation at these strings
    stop=["END", "---"],
    
    # Presence penalty: -2 to 2, discourages repetition of topics
    presence_penalty=0,
    
    # Frequency penalty: -2 to 2, discourages repetition of words
    frequency_penalty=0
)
