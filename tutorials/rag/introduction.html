RAG — Retrieval-Augmented Generation

Learn why RAG exists, how it works end-to-end, and build a complete RAG pipeline from scratch with embeddings, vector stores, and LLMs.

📋 Table of Contents

1. Why RAG?

LLMs have a knowledge cutoff and can't access private data. RAG (Retrieval-Augmented Generation) solves this by retrieving relevant documents at query time and injecting them into the prompt — giving the model up-to-date, domain-specific knowledge without fine-tuning.

  • ✅ No training required — add documents any time
  • ✅ Cites sources — traceable and auditable answers
  • ✅ Works with private data — never leaves your system
  • ✅ Cheaper than fine-tuning — no GPU costs
  • ❌ Quality depends on retrieval quality
  • ❌ Limited by context window size

2. Embeddings & Similarity

An embedding is a vector (array of floats) representing the semantic meaning of text. Similar meanings → similar vectors → high cosine similarity.

Python — compute embeddings and similarity
from openai import OpenAI
import numpy as np

client = OpenAI()

def embed(text: str) -> np.ndarray:
    res = client.embeddings.create(model="text-embedding-3-small", input=text)
    return np.array(res.data[0].embedding)

def cosine_similarity(a: np.ndarray, b: np.ndarray) -> float:
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# Test semantic similarity
sentences = [
    "Python is a programming language.",
    "Python is used for coding software.",  # very similar
    "I love eating pizza.",                  # unrelated
]

embeddings = [embed(s) for s in sentences]

print(f"Same topic:  {cosine_similarity(embeddings[0], embeddings[1]):.3f}")  # ~0.95
print(f"Diff topic:  {cosine_similarity(embeddings[0], embeddings[2]):.3f}")  # ~0.20

3. Vector Stores

A vector store indexes embeddings for fast nearest-neighbor search. Common options: FAISS (local), Chroma (local), Pinecone (cloud), Weaviate (cloud).

Python — FAISS vector store from scratch
import faiss
import numpy as np
from openai import OpenAI

client = OpenAI()

documents = [
    "Python supports object-oriented programming with classes and inheritance.",
    "Django is a high-level Python web framework for rapid development.",
    "FastAPI is an async Python framework for building APIs with type hints.",
    "NumPy provides efficient array operations and mathematical functions.",
    "Pandas is used for data manipulation and analysis with DataFrames.",
]

# Build the index
def get_embeddings(texts):
    res = client.embeddings.create(model="text-embedding-3-small", input=texts)
    return np.array([r.embedding for r in res.data], dtype="float32")

vectors = get_embeddings(documents)
dim = vectors.shape[1]

index = faiss.IndexFlatIP(dim)        # Inner Product = cosine sim on normalized vecs
faiss.normalize_L2(vectors)
index.add(vectors)

# Query
query = "What framework should I use to build a REST API?"
q_vec = get_embeddings([query])
faiss.normalize_L2(q_vec)

scores, indices = index.search(q_vec, k=2)
for score, idx in zip(scores[0], indices[0]):
    print(f"[{score:.3f}] {documents[idx]}")

4. Complete RAG Pipeline

Python — end-to-end RAG with LangChain
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_community.vectorstores import FAISS
from langchain_core.documents import Document
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough

# 1. Your knowledge base
docs = [
    Document(page_content="FastAPI is a modern web framework for Python. Use uvicorn to run it.", metadata={"source": "fastapi_docs"}),
    Document(page_content="To install FastAPI: pip install fastapi uvicorn", metadata={"source": "fastapi_docs"}),
    Document(page_content="Django uses the MVC pattern and comes with an admin interface.", metadata={"source": "django_docs"}),
    Document(page_content="Flask is a lightweight WSGI web framework for Python.", metadata={"source": "flask_docs"}),
]

# 2. Create vector store
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
vectorstore = FAISS.from_documents(docs, embeddings)
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

# 3. RAG prompt
prompt = ChatPromptTemplate.from_template("""Answer using ONLY the provided context.
If the answer is not in the context, say "I do not have that information."

Context:
{context}

Question: {question}""")

# 4. Build the chain
llm = ChatOpenAI(model="gpt-4o-mini")

def format_docs(docs):
    return "\n\n".join(
        f"[{d.metadata.get('source', 'unknown')}] {d.page_content}"
        for d in docs
    )

rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

# 5. Query it
answer = rag_chain.invoke("How do I run a FastAPI app?")
print(answer)

5. Evaluating RAG Quality

Python — faithfulness and relevance scores
def evaluate_rag(question: str, context: str, answer: str) -> dict:
    """Evaluate RAG output on faithfulness and relevance."""
    llm = ChatOpenAI(model="gpt-4o-mini")

    faithfulness_prompt = f"""Is this answer supported by the context?
Context: {context}
Answer: {answer}
Reply with a score 1-5 and one sentence reason. Format: SCORE: X | REASON: ..."""

    relevance_prompt = f"""Does this answer actually address the question?
Question: {question}
Answer: {answer}
Reply with a score 1-5 and one sentence reason. Format: SCORE: X | REASON: ..."""

    faith_result = llm.invoke(faithfulness_prompt).content
    rel_result = llm.invoke(relevance_prompt).content

    return {"faithfulness": faith_result, "relevance": rel_result}

# Use it after each RAG response
q = "How do I run a FastAPI app?"
ctx = "FastAPI uses uvicorn as its ASGI server."
ans = rag_chain.invoke(q)
scores = evaluate_rag(q, ctx, ans)
print(scores)
