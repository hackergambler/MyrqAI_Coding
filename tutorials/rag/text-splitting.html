Text Splitting Strategies

Choose the right splitting strategy for your RAG pipeline: recursive, character, semantic, token-aware, and code-specific splitters.

Why Splitting Matters

If chunks are too large → they use too many context tokens and contain irrelevant text. Too small → they lose context and meaning. The right chunk size for your use case is the single biggest factor in RAG quality.

💡 Rule of Thumb

Start with chunk_size=1000, chunk_overlap=200. For technical docs, smaller (500/100) works better. For narrative text, larger (1500/300) preserves more context.

RecursiveCharacterTextSplitter

The go-to splitter. Tries to split on paragraphs → sentences → words → characters in that order, keeping chunks as semantically intact as possible.

Python — recursive text splitting
from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,        # target chunk size in characters
    chunk_overlap=200,       # overlap to preserve context between chunks
    length_function=len,      # or use token counting (see below)
    separators=["\n\n", "\n", ". ", " ", ""],  # split hierarchy
    add_start_index=True,    # add char offset to metadata
    strip_whitespace=True,
)

# Split documents
from langchain_core.documents import Document
doc = Document(page_content="..." * 100)  # your long doc
chunks = splitter.split_documents([doc])
print(f"{len(chunks)} chunks created")
for chunk in chunks[:3]:
    print(f"  [{chunk.metadata.get('start_index',0)}] {len(chunk.page_content)} chars")

Token-Based Splitting

Split by actual token count (not characters). Critical for staying within model context limits.

Python — token-aware splitting
import tiktoken
from langchain_text_splitters import RecursiveCharacterTextSplitter

# Use OpenAI's tokenizer
enc = tiktoken.encoding_for_model("gpt-4o")

def token_count(text: str) -> int:
    return len(enc.encode(text))

token_splitter = RecursiveCharacterTextSplitter(
    chunk_size=512,          # 512 tokens per chunk
    chunk_overlap=64,         # 64 token overlap
    length_function=token_count,
)

# Or use the built-in TokenTextSplitter
from langchain_text_splitters import TokenTextSplitter
token_splitter2 = TokenTextSplitter(
    model_name="gpt-4o",
    chunk_size=512,
    chunk_overlap=64
)

Code-Aware Splitting

Python — language-aware code splitting
from langchain_text_splitters import RecursiveCharacterTextSplitter, Language

# Python-aware: splits on class/function boundaries first
python_splitter = RecursiveCharacterTextSplitter.from_language(
    language=Language.PYTHON,
    chunk_size=1000,
    chunk_overlap=100
)

python_code = """
class DataProcessor:
    def __init__(self, data):
        self.data = data

    def process(self):
        return [x * 2 for x in self.data]

    def filter_evens(self):
        return [x for x in self.data if x % 2 == 0]

def standalone_function(x, y):
    return x + y
"""

chunks = python_splitter.create_documents([python_code])
for chunk in chunks:
    print("---")
    print(chunk.page_content)

# Other supported languages
js_splitter = RecursiveCharacterTextSplitter.from_language(Language.JS, chunk_size=500)
md_splitter = RecursiveCharacterTextSplitter.from_language(Language.MARKDOWN, chunk_size=800)
html_splitter = RecursiveCharacterTextSplitter.from_language(Language.HTML, chunk_size=600)

Semantic Splitting

Split based on semantic similarity, keeping related sentences together even across paragraph breaks.

Python — semantic chunking with embeddings
from langchain_experimental.text_splitter import SemanticChunker
from langchain_openai import OpenAIEmbeddings

# Splits when semantic similarity drops below threshold
semantic_splitter = SemanticChunker(
    OpenAIEmbeddings(),
    breakpoint_threshold_type="percentile",  # or "standard_deviation", "interquartile"
    breakpoint_threshold_amount=95             # 95th percentile breakpoints
)

long_text = """Python is a versatile programming language known for its readability.
It was created by Guido van Rossum and first released in 1991.

Machine learning is transforming how we build software.
Models like GPT-4 can write code, analyze data, and answer questions.

The solar system contains eight planets. Mars is the fourth planet from the Sun."""

chunks = semantic_splitter.create_documents([long_text])
print(f"{len(chunks)} semantic chunks")
for c in chunks:
    print(f"\n[Chunk] {c.page_content[:100]}")

Metadata Enrichment

Python — adding metadata to chunks
def enrich_chunks(docs, splitter, source_name: str) -> list:
    """Split docs and add rich metadata to each chunk."""
    chunks = splitter.split_documents(docs)
    for i, chunk in enumerate(chunks):
        chunk.metadata.update({
            "chunk_id": i,
            "source": source_name,
            "chunk_size": len(chunk.page_content),
            "total_chunks": len(chunks),
        })
    return chunks

from langchain_community.document_loaders import PyPDFLoader
loader = PyPDFLoader("manual.pdf")
docs = loader.load()
chunks = enrich_chunks(docs, splitter, "product_manual")
print(chunks[0].metadata)
