System Prompts — Persona, Rules & Constraints

Master system prompts: set personas, enforce output formats, add guardrails, and create consistent AI assistants.

What Are System Prompts?

A system prompt is instructions given to the LLM before the user speaks. It sets the persona, rules, output format, and constraints that apply to every response. It's your most powerful tool for controlling LLM behavior.

Python - basic system prompt patterns
from openai import OpenAI
client = OpenAI()

def ask(system: str, user: str) -> str:
    resp = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role":"system","content":system},
            {"role":"user","content":user}
        ]
    )
    return resp.choices[0].message.content

# 1. Persona assignment
print(ask(
    "You are a senior Python engineer at Google. You give expert advice with production-ready code.",
    "How should I handle errors in async code?"
))

# 2. Format enforcement
print(ask(
    "You are a technical writer. Always respond with: SUMMARY (1 sentence), DETAILS (3 bullets), CODE (if applicable).",
    "Explain Python context managers."
))

# 3. Constraints and rules
print(ask(
    "You are a customer support agent for PyShop. Rules: 1) Never reveal pricing without authorization. 2) Always apologize first. 3) Escalate billing issues to billing@pyshop.com.",
    "Why was I charged twice?"
))

The RICE Framework for System Prompts

  • Role — who the AI is (You are an expert Python engineer...)
  • Instructions — what it should do (Always include type hints...)
  • Constraints — what it must NOT do (Never output untested code...)
  • Examples — show the exact output format you want
Python - production-quality system prompt using RICE
SYSTEM = """You are CodeReview-AI, a senior software engineer specializing in Python.

ROLE: Expert code reviewer with 10+ years Python experience.

INSTRUCTIONS:
- Review code for: correctness, security, performance, readability
- Always show the improved version after your critique
- Use Python type hints in all code examples
- Reference PEP 8 and PEP 484 where relevant

CONSTRAINTS:
- Never say a solution "looks fine" without checking edge cases
- Never omit error handling in production code examples
- Do not suggest third-party libraries without noting install command

FORMAT:
Issues Found:
1. [Issue] — [Why it matters]

Improved Code:
```python
[improved code here]
```
"""

review = ask(SYSTEM, "def get_user(id): return db.query(f'SELECT * FROM users WHERE id={id}')")
print(review)
💡 System prompt length

System prompts can be thousands of tokens. Use them to encode detailed instructions, examples, and business rules. This is far cheaper than including instructions in every user message — especially with prompt caching.
