Prompt Engineering Fundamentals

Write prompts that consistently get the output you want. Learn the core techniques: role assignment, output format control, few-shot examples, and chain-of-thought.

The Anatomy of a Great Prompt

Every effective prompt has 4 components: Role (who is the AI?), Context (what background is needed?), Task (what exactly should it do?), and Format (how should it respond?).

Python - bad vs good prompt comparison
from openai import OpenAI
client = OpenAI()

def ask(system: str, user: str) -> str:
    resp = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "system", "content": system},
                  {"role": "user", "content": user}],
        max_tokens=300
    )
    return resp.choices[0].message.content

# ❌ Bad: vague, no format, no context
bad = ask("You are helpful.", "Tell me about Python.")

# ✅ Good: role + context + task + format
good = ask(
    """You are a senior Python engineer with 10 years experience.
Your audience is developers who know JavaScript but are new to Python.
Always give concrete code examples. Be concise - max 3 paragraphs.""",
    """Compare Python lists vs JavaScript arrays.
Focus on: mutability, methods, and performance differences.
Format as: Similarities, Key Differences, Code Example."""
)
print("GOOD:\n", good)

Output Format Control

Python - force specific output formats
# Force JSON output
json_response = ask(
    "Return ONLY valid JSON. No markdown, no prose, no code fences.",
    "Extract: name=Alice, role=Engineer, years=5, skills=Python,AWS,Docker"
)
import json
data = json.loads(json_response)

# Force a specific structure
structured = ask(
    """Respond EXACTLY in this format, no deviations:
SUMMARY: [one sentence]
PROS:
- [point 1]
- [point 2]  
- [point 3]
CONS:
- [point 1]
- [point 2]
VERDICT: [one word: Recommended/Avoid/Neutral]""",
    "Evaluate: using SQLite for a production web app with 10k daily users"
)
print(structured)

# Force numbered list
steps = ask(
    "Respond ONLY with a numbered list. Each step on its own line. No prose.",
    "How do I deploy a FastAPI app to AWS Lambda?"
)

Few-Shot Prompting

Python - teach by example
# Show 3 examples, then ask for the real thing
few_shot_prompt = """Convert informal text to formal business English.

Informal: "hey can u send the report asap its rly urgent"
Formal: "Could you please send the report at your earliest convenience? This matter is time-sensitive."

Informal: "the meeting got pushed back cuz john cant make it"  
Formal: "The meeting has been rescheduled due to John's unavailability."

Informal: "we need to fix this bug its breaking everything"
Formal: "We need to address this critical bug immediately, as it is causing significant system disruption."

Informal: {text}
Formal:"""

def formalize(text: str) -> str:
    resp = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": few_shot_prompt.format(text=text)}],
        max_tokens=100
    )
    return resp.choices[0].message.content.strip()

print(formalize("gonna be late 2 the standup sry"))

Chain-of-Thought

Python - force step-by-step reasoning
# CoT dramatically improves accuracy on complex problems
cot_system = """Solve problems by thinking step by step.
Always structure your response as:
Step 1: [first step]
Step 2: [second step]
...
Therefore: [final answer]"""

result = ask(cot_system, """A store sells apples for $0.50 each and oranges for $0.75 each.
Alice buys 6 apples and 4 oranges. She pays with a $10 bill.
How much change does she receive?""")
print(result)

# Zero-shot CoT: just add "Let's think step by step"
zero_shot_cot = ask(
    "You are a logical reasoner.",
    "If all Bloops are Razzies, and all Razzies are Lazzies, are all Bloops definitely Lazzies? Let's think step by step."
)
💡 Magic Phrase

Adding "Let's think step by step" or "Think carefully before answering" to any prompt significantly improves accuracy on reasoning tasks — even without few-shot examples.
