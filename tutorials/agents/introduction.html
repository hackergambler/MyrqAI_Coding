Introduction to AI Agents

Agents are LLMs that can take actions — calling tools, running code, browsing the web. Learn the ReAct loop, tool design principles, and build your first agent from scratch.

What Makes an Agent?

A chatbot responds to a single message. An agent autonomously plans and executes a sequence of actions to complete a goal. The core loop is: Think → Act → Observe → Repeat.

  • LLM — the reasoning engine (GPT-4o, Claude, Gemini)
  • Tools — functions the LLM can call (web search, code execution, APIs)
  • Memory — conversation history + vector stores for long-term context
  • Orchestrator — the loop that connects them all
ℹ️ ReAct Pattern

ReAct = Reasoning + Acting. The LLM writes a Thought, then an Action, observes the Result, and repeats until it has a final answer. This is the foundation of most production agents.

Your First ReAct Agent from Scratch

Python - minimal ReAct agent
from openai import OpenAI
import json, re

client = OpenAI()

# Tools
def search(query: str) -> str:
    return f"Search results for '{query}': [Python 3.12 released Oct 2023, adds type parameter syntax]"

def calculate(expression: str) -> str:
    try: return str(eval(expression, {"__builtins__": {}}))
    except Exception as e: return f"Error: {e}"

TOOLS = {"search": search, "calculate": calculate}

SYSTEM = """You are an agent. Solve tasks step-by-step using this format:

Thought: [your reasoning]
Action: tool_name
Action Input: the input to the tool

After seeing a result:
Observation: [tool result]
... repeat as needed ...

Final Answer: [your answer]

Available tools: search, calculate"""

def run_agent(task: str, max_steps: int = 8) -> str:
    messages = [
        {"role": "system", "content": SYSTEM},
        {"role": "user", "content": task}
    ]
    for step in range(max_steps):
        resp = client.chat.completions.create(
            model="gpt-4o-mini", messages=messages,
            stop=["Observation:"]  # stop before hallucinating the result
        )
        text = resp.choices[0].message.content
        messages.append({"role": "assistant", "content": text})

        if "Final Answer:" in text:
            return text.split("Final Answer:")[1].strip()

        # Parse action
        action_match = re.search(r"Action: (\w+)
Action Input: (.+)", text, re.DOTALL)
        if not action_match: break
        tool_name, tool_input = action_match.group(1), action_match.group(2).strip()

        result = TOOLS.get(tool_name, lambda x: "Tool not found")(tool_input)
        messages.append({"role": "user", "content": f"Observation: {result}"})
        print(f"Step {step+1}: {tool_name}({tool_input[:40]}) → {result[:60]}")

    return "Max steps reached"

print(run_agent("What is the square root of the year Python 3.12 was released?"))
💡 Stop Sequences

Using stop=["Observation:"] prevents the LLM from hallucinating tool results. Always let your code provide the observation.
