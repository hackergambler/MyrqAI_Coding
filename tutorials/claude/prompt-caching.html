Claude Prompt Caching

Cache large system prompts and context once, then reuse them across requests — slashing token costs by up to 90% and reducing time-to-first-token on repeated calls.

In This Lesson

  1. What Is Prompt Caching?
  2. How It Works
  3. Basic Example
  4. Placing Cache Breakpoints
  5. Real-World Use Cases
  6. Hands-On Exercise

What Is Prompt Caching?

Prompt caching lets you store a prefix of your prompt (system instructions, documents, few-shot examples) on Anthropic's servers. On subsequent requests that share the same prefix, Claude skips reprocessing those tokens — delivering faster responses and dramatically lower costs.

90%
Cost reduction on cached tokens
85%
Latency reduction for large prompts
When To Use It

Prompt caching pays off when you have a large, static context (system prompt, reference docs, examples) that you reuse across many requests. The minimum cacheable prefix is 1,024 tokens for Claude Haiku and 2,048 tokens for Sonnet/Opus.

How It Works

You mark cache breakpoints in your request using cache_control blocks. When Claude processes the request, it checks whether those prefix tokens are already cached:

  • Cache miss — normal input token price; result is stored for 5 minutes (ephemeral).
  • Cache hit — cached token price (~10% of normal), no reprocessing.

Cache TTL is 5 minutes by default. Every cache hit refreshes the TTL, so active conversations stay warm automatically.

Cache Write Cost

Writing to the cache costs 25% more than a normal input token. You break even after just 2 cache hits, so for any prompt called 3+ times the math strongly favors caching.

Basic Example

To enable caching, set betas=["prompt-caching-2024-07-31"] and add "cache_control": {"type": "ephemeral"} to the content block you want cached.

python
import anthropic

client = anthropic.Anthropic()

# Large system prompt — only processed once, then cached
SYSTEM_PROMPT = """
You are an expert Python tutor with deep knowledge of data structures,
algorithms, and best practices. When answering questions you always:
1. Explain the concept clearly
2. Provide a working code example
3. Highlight common pitfalls
4. Suggest further reading

[... imagine this is 3000+ tokens of detailed instructions ...]
"""

def ask_claude(user_question: str) -> str:
    response = client.beta.messages.create(
        model="claude-opus-4-6",
        max_tokens=1024,
        # Enable the prompt caching beta
        betas=["prompt-caching-2024-07-31"],
        system=[
            {
                "type": "text",
                "text": SYSTEM_PROMPT,
                # Mark this block as cacheable
                "cache_control": {"type": "ephemeral"}
            }
        ],
        messages=[{"role": "user", "content": user_question}]
    )
    return response.content[0].text

# First call: cache miss — system prompt is processed + cached
answer1 = ask_claude("Explain binary search trees")

# Second call: cache hit — system prompt costs 90% less
answer2 = ask_claude("What are Python decorators?")

Placing Cache Breakpoints

You can have up to 4 cache breakpoints per request. Each breakpoint caches everything from the start of the prompt up to that point. Place them strategically:

python — multi-block caching
response = client.beta.messages.create(
    model="claude-opus-4-6",
    max_tokens=2048,
    betas=["prompt-caching-2024-07-31"],
    system=[
        # Block 1: static role definition (always the same)
        {
            "type": "text",
            "text": "You are a senior software engineer...",
            "cache_control": {"type": "ephemeral"}
        },
        # Block 2: per-session document (changes per user session)
        {
            "type": "text",
            "text": codebase_context,   # injected at session start
            "cache_control": {"type": "ephemeral"}
        }
    ],
    messages=conversation_history   # grows per turn
)
Caching Conversation History

You can also cache the messages array up to the last user turn. Add cache_control to the final assistant message in history. This is useful for long multi-turn conversations where earlier turns never change.

Real-World Use Cases

1. Document Q&A

Load a large PDF or codebase into the system prompt once. All follow-up questions hit the cache — turning a slow, expensive operation into a fast, cheap one.

python — document Q&A
with open("technical_spec.txt") as f:
    document = f.read()   # Could be 10,000+ tokens

def query_document(question):
    return client.beta.messages.create(
        model="claude-opus-4-6",
        max_tokens=1024,
        betas=["prompt-caching-2024-07-31"],
        system=[{
            "type": "text",
            "text": f"Answer questions using this document:\n\n{document}",
            "cache_control": {"type": "ephemeral"}
        }],
        messages=[{"role": "user", "content": question}]
    ).content[0].text

# All 3 queries share one cached document — pay for the document once
query_document("What are the rate limits?")
query_document("Summarise the authentication section")
query_document("List all API endpoints")

2. Checking Cache Usage

The response's usage object tells you exactly how many tokens were read from cache vs. freshly processed:

python — inspect cache stats
response = client.beta.messages.create( ... )

usage = response.usage
print(f"Input tokens:          {usage.input_tokens}")
print(f"Cache write tokens:    {usage.cache_creation_input_tokens}")
print(f"Cache read tokens:     {usage.cache_read_input_tokens}")
print(f"Output tokens:         {usage.output_tokens}")

# Calculate real savings
normal_cost   = (usage.input_tokens + usage.cache_creation_input_tokens) * 0.000015
cached_cost   = usage.cache_read_input_tokens * 0.0000015  # 10x cheaper
print(f"Savings: ${normal_cost - cached_cost:.4f}")

Hands-On Exercise

Build a simple CLI chatbot that caches its system prompt and logs cache hit/miss stats after each turn.

exercise.py
import anthropic

client = anthropic.Anthropic()

SYSTEM = """
You are a concise Python expert. Keep answers under 150 words.
Always include a one-line code snippet when relevant.
Use plain language — no jargon without explanation.
"""  # Pad this to >1024 tokens in real usage

history = []
cache_hits = 0
cache_misses = 0

print("Python Expert Chatbot (type 'quit' to exit)\n")

while True:
    user_input = input("You: ")
    if user_input.lower() == "quit":
        break

    history.append({"role": "user", "content": user_input})

    response = client.beta.messages.create(
        model="claude-haiku-4-5-20251001",  # cheapest model
        max_tokens=300,
        betas=["prompt-caching-2024-07-31"],
        system=[{"type": "text", "text": SYSTEM,
                 "cache_control": {"type": "ephemeral"}}],
        messages=history
    )

    reply = response.content[0].text
    history.append({"role": "assistant", "content": reply})

    u = response.usage
    if u.cache_read_input_tokens > 0:
        cache_hits += 1
        status = f"[CACHE HIT — {u.cache_read_input_tokens} tokens saved]"
    else:
        cache_misses += 1
        status = f"[CACHE MISS — {u.cache_creation_input_tokens} tokens written]"

    print(f"\nClaude: {reply}")
    print(f"{status} | Hits: {cache_hits}  Misses: {cache_misses}\n")
Expected Output

First message: CACHE MISS — N tokens written. Every subsequent message: CACHE HIT — N tokens saved. The system prompt is only processed once per 5-minute window.
