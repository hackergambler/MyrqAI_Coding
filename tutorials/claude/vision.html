Claude Vision: Image Analysis & Multimodal AI

How Claude Vision Works

Claude's vision capability lets you include images directly in the messages array. Claude understands both the image content and any accompanying text, enabling rich multimodal reasoning.

Images can be passed in two ways:

  • URL source — Claude fetches and reads the image from a public URL.
  • Base64 source — You encode the image bytes directly in the request payload, ideal for local files or private images.

Supported formats: image/jpeg, image/png, image/gif, image/webp. Maximum image size is 5 MB per image, with up to 20 images per request.

Which Models Support Vision?

All Claude 3+ models (Haiku, Sonnet, Opus) and Claude 3.5/3.7 variants support image input. Always use claude-sonnet-4-6 or later for best accuracy on complex visual reasoning tasks.

Sending an Image via URL

The simplest approach — pass a publicly accessible image URL in the source block:

python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "image",
                    "source": {
                        "type": "url",
                        "url": "https://upload.wikimedia.org/wikipedia/commons/thumb/4/47/PNG_transparency_demonstration_1.png/280px-PNG_transparency_demonstration_1.png",
                    },
                },
                {
                    "type": "text",
                    "text": "Describe what you see in this image."
                }
            ],
        }
    ],
)

print(response.content[0].text)
Tip

Always place the image block before the text block in the content array so Claude reads the image in context when it processes your question.

Sending an Image via Base64

For local or private images, encode the file as base64 and include it in the request:

python
import anthropic
import base64
from pathlib import Path

client = anthropic.Anthropic()

# Load and encode a local image
image_path = Path("screenshot.png")
image_data = base64.standard_b64encode(image_path.read_bytes()).decode("utf-8")

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "image",
                    "source": {
                        "type": "base64",
                        "media_type": "image/png",
                        "data": image_data,
                    },
                },
                {
                    "type": "text",
                    "text": "What UI elements are visible in this screenshot?"
                }
            ],
        }
    ],
)

print(response.content[0].text)

OCR: Extracting Text from Images

Claude can read text embedded in images — receipts, documents, screenshots, whiteboards — with high accuracy. Ask it to extract text precisely:

python
import anthropic
import base64

client = anthropic.Anthropic()

def extract_text_from_image(image_path: str) -> str:
    with open(image_path, "rb") as f:
        image_data = base64.standard_b64encode(f.read()).decode("utf-8")

    # Detect media type from extension
    ext = image_path.rsplit(".", 1)[-1].lower()
    media_types = {"jpg": "image/jpeg", "jpeg": "image/jpeg", "png": "image/png", "webp": "image/webp"}
    media_type = media_types.get(ext, "image/jpeg")

    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=2048,
        messages=[{
            "role": "user",
            "content": [
                {"type": "image", "source": {"type": "base64", "media_type": media_type, "data": image_data}},
                {"type": "text", "text": "Extract all text from this image exactly as it appears. Preserve line breaks and formatting."}
            ]
        }]
    )
    return response.content[0].text

# Usage
text = extract_text_from_image("receipt.jpg")
print(text)

Multi-Image Comparisons

Pass multiple images in a single message to have Claude compare, contrast, or analyze them together:

python
import anthropic
import base64

client = anthropic.Anthropic()

def load_image(path: str) -> dict:
    with open(path, "rb") as f:
        data = base64.standard_b64encode(f.read()).decode()
    return {
        "type": "image",
        "source": {"type": "base64", "media_type": "image/png", "data": data}
    }

# Compare two UI mockups
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{
        "role": "user",
        "content": [
            load_image("mockup_v1.png"),
            load_image("mockup_v2.png"),
            {
                "type": "text",
                "text": "Compare these two UI mockups. What changed between v1 and v2? Which version has better UX and why?"
            }
        ]
    }]
)

print(response.content[0].text)
Cost Awareness

Each image uses tokens based on its dimensions. A 1092×1092 image costs roughly 1,590 input tokens. For high-volume pipelines, use smaller or resized images to reduce costs.

Extracting Structured Data from Images

Combine vision with a system prompt to extract structured JSON from images — great for parsing receipts, invoices, forms, or charts:

python
import anthropic
import base64
import json

client = anthropic.Anthropic()

def parse_receipt(image_path: str) -> dict:
    with open(image_path, "rb") as f:
        image_data = base64.standard_b64encode(f.read()).decode()

    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        system="""You are a receipt parser. Extract data as JSON only.
Return a JSON object with keys: store_name, date, items (list of {name, price}), subtotal, tax, total.
Do not include any text outside the JSON object.""",
        messages=[{
            "role": "user",
            "content": [
                {"type": "image", "source": {"type": "base64", "media_type": "image/jpeg", "data": image_data}},
                {"type": "text", "text": "Parse this receipt."}
            ]
        }]
    )

    raw = response.content[0].text.strip()
    return json.loads(raw)

# Usage
receipt = parse_receipt("receipt.jpg")
print(f"Total: ${receipt['total']}")
for item in receipt["items"]:
    print(f"  {item['name']}: ${item['price']}")

Hands-on Exercise

Build a screenshot accessibility auditor. Given a UI screenshot, use Claude to identify potential accessibility issues (missing alt text areas, low contrast sections, small tap targets) and return them as a structured list.

Requirements

  1. Accept a local PNG/JPG screenshot via command-line argument.
  2. Send it to Claude with a prompt instructing it to audit for WCAG accessibility issues.
  3. Print each issue with: issue type, location description, and suggested fix.
  4. Add a --json flag that outputs the full result as JSON instead of plain text.

Starter Code

python — accessibility_auditor.py
import anthropic
import base64
import argparse
import json

client = anthropic.Anthropic()

SYSTEM = """You are a WCAG 2.1 accessibility auditor.
Analyze the provided UI screenshot and identify accessibility issues.
Return a JSON array of issues, each with:
  - type: one of [contrast, tap_target, missing_label, keyboard_nav, other]
  - location: human-readable description of where the issue is
  - fix: a short, actionable fix suggestion
Return only the JSON array, no other text."""

def audit(image_path: str) -> list:
    with open(image_path, "rb") as f:
        data = base64.standard_b64encode(f.read()).decode()
    ext = image_path.rsplit(".", 1)[-1].lower()
    media_type = "image/png" if ext == "png" else "image/jpeg"

    resp = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        system=SYSTEM,
        messages=[{
            "role": "user",
            "content": [
                {"type": "image", "source": {"type": "base64", "media_type": media_type, "data": data}},
                {"type": "text", "text": "Audit this UI screenshot for accessibility issues."}
            ]
        }]
    )
    return json.loads(resp.content[0].text)

def main():
    parser = argparse.ArgumentParser(description="Audit a UI screenshot for accessibility issues")
    parser.add_argument("image", help="Path to screenshot (PNG or JPG)")
    parser.add_argument("--json", action="store_true", help="Output as JSON")
    args = parser.parse_args()

    issues = audit(args.image)

    if args.json:
        print(json.dumps(issues, indent=2))
    else:
        print(f"Found {len(issues)} accessibility issue(s):\n")
        for i, issue in enumerate(issues, 1):
            print(f"{i}. [{issue['type'].upper()}] {issue['location']}")
            print(f"   Fix: {issue['fix']}\n")

if __name__ == "__main__":
    main()

Real-World Use Cases

Document Intelligence

Parse invoices, contracts, and forms — extract line items, dates, and totals into structured records automatically.

UI QA Automation

Screenshot your app after deployments and have Claude flag regressions, layout breaks, or missing elements.

Content Moderation

Detect inappropriate images, verify image-text alignment, or flag visual policy violations at scale.

Diagram Analysis

Extract data from charts, flowcharts, or architecture diagrams to power downstream summarization or Q&A.

Performance Tip

For repeated analysis of the same base image with different questions, use prompt caching to cache the image block. This cuts cost by up to 90% and reduces latency significantly.
