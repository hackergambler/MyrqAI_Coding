1. Anatomy of a Message

Every call to the Claude Messages API has the same skeleton. Understanding each field is the foundation for everything else.

Python — minimal call
import anthropic

client = anthropic.Anthropic(api_key="sk-ant-...")

response = client.messages.create(
    model="claude-opus-4-5",          # model ID
    max_tokens=1024,                  # required upper bound on tokens
    messages=[
        {"role": "user", "content": "Explain async/await in Python in 3 lines."}
    ]
)

print(response.content[0].text)     # text of the first content block
print(response.usage)               # input_tokens, output_tokens
print(response.stop_reason)         # "end_turn" | "max_tokens" | "stop_sequence"

Response Object Fields

  • response.id — unique message ID (for logging)
  • response.model — model that responded
  • response.role — always "assistant"
  • response.content — list of content blocks (text / tool_use)
  • response.stop_reason — why generation stopped
  • response.usage.input_tokens — tokens in your prompt
  • response.usage.output_tokens — tokens in the response
Python — inspect full response
response = client.messages.create(
    model="claude-haiku-4-5-20251001",
    max_tokens=256,
    messages=[{"role": "user", "content": "What is 2+2?"}]
)

print(f"ID: {response.id}")
print(f"Model: {response.model}")
print(f"Stop: {response.stop_reason}")
print(f"In: {response.usage.input_tokens} Out: {response.usage.output_tokens}")
for block in response.content:
    if block.type == "text":
        print("Text:", block.text)

2. System Prompts

The system parameter sets Claude's persona, constraints, and context before the user speaks. Unlike OpenAI's system message inside the array, Claude uses a dedicated top-level parameter — cleaner and more explicit.

Python — role-specific system prompts
# 1. Coding assistant persona
coding_bot = client.messages.create(
    model="claude-opus-4-5",
    max_tokens=512,
    system="""You are an expert Python engineer. Always include: type annotations, docstrings, and a usage example.""",
    messages=[{"role": "user", "content": "Write a function to flatten a nested list."}]
)

# 2. JSON-only API persona
json_bot = client.messages.create(
    model="claude-haiku-4-5-20251001",
    max_tokens=256,
    system="""You are a data extraction API. Always respond with valid JSON only.""",
    messages=[{"role": "user", "content": "Extract: name=Alice, age=30, city=London"}]
)
import json
data = json.loads(json_bot.content[0].text)
print(data)  # {'name': 'Alice', 'age': 30, 'city': 'London'}

# 3. Customer support persona with guardrails
support_bot = client.messages.create(
    model="claude-opus-4-5",
    max_tokens=512,
    system="""You are a support agent for AcmeSaaS. Only discuss AcmeSaaS products. Never reveal internal pricing. Always end with a follow-up offer.""",
    messages=[{"role": "user", "content": "How does your product compare to Competitor X?"}]
)
💡 System Prompt Length

System prompts can be thousands of tokens. Use them to encode detailed instructions, examples, and business rules. This is much cheaper than including instructions in every user message.

3. Multi-turn Conversations

Claude has no built-in memory. You are responsible for sending the entire conversation history on every request. The messages array alternates user / assistant and must start with user.

Python — stateful chat class
from anthropic import Anthropic

class ChatSession:
    def __init__(self, system: str = ""):
        self.client = Anthropic()
        self.system = system
        self.history: list[dict] = []

    def chat(self, user_message: str) -> str:
        self.history.append({"role": "user", "content": user_message})
        kwargs = {
            "model": "claude-opus-4-5",
            "max_tokens": 1024,
            "messages": self.history,
        }
        if self.system:
            kwargs["system"] = self.system
        response = self.client.messages.create(**kwargs)
        assistant_text = response.content[0].text
        self.history.append({"role": "assistant", "content": assistant_text})
        return assistant_text

    def reset(self):
        self.history = []

# Usage
session = ChatSession(system="You are a Python tutor.")
print(session.chat("What is a generator?"))
print(session.chat("Show me a real-world example."))   # remembers context
print(session.chat("How is it different from a list?"))  # still in context
Python — trimming history to stay in budget
def trim_history(history: list[dict], max_turns: int = 20) -> list[dict]:
    """Keep only the last N user+assistant pairs."""
    if len(history) <= max_turns * 2:
        return history
    # Always keep first message for context, trim the middle
    return [history[0]] + history[-(max_turns * 2 - 1):]

# Or trim by token estimate
def trim_by_tokens(history: list[dict], max_tokens: int = 50_000) -> list[dict]:
    trimmed = []
    total = 0
    for msg in reversed(history):
        tokens = len(msg["content"].split()) * 4 // 3  # rough estimate
        if total + tokens > max_tokens:
            break
        trimmed.insert(0, msg)
        total += tokens
    return trimmed

4. Streaming Responses

For long responses or real-time UIs, streaming lets you process tokens as they arrive instead of waiting for the full response. Use the stream=True flag or the stream() context manager.

Python — basic streaming
import anthropic

client = anthropic.Anthropic()

with client.messages.stream(
    model="claude-opus-4-5",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Write a 200-word story about a robot."}]
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)

# Access final message after streaming
final = stream.get_final_message()
print(f"\nTokens used: {final.usage.input_tokens} in / {final.usage.output_tokens} out")
Python — async streaming for web apps
import asyncio
import anthropic

async def stream_response(prompt: str):
    client = anthropic.AsyncAnthropic()
    async with client.messages.stream(
        model="claude-opus-4-5",
        max_tokens=1024,
        messages=[{"role": "user", "content": prompt}]
    ) as stream:
        async for text in stream.text_stream:
            yield text  # async generator for FastAPI / websockets

# FastAPI streaming endpoint
from fastapi import FastAPI
from fastapi.responses import StreamingResponse

app = FastAPI()

@app.get("/stream")
async def stream_endpoint(q: str):
    return StreamingResponse(
        stream_response(q),
        media_type="text/event-stream"
    )

5. Prefill (Assistant Turn Injection)

You can start Claude's response by providing an incomplete assistant turn as the last message. Claude will continue from exactly where you left off — useful for forcing output format, skipping preambles, or filling templates.

Python — prefill to force JSON output
# Without prefill: Claude might say "Sure! Here's the JSON:"
# With prefill: Claude starts directly with {

response = client.messages.create(
    model="claude-haiku-4-5-20251001",
    max_tokens=256,
    messages=[
        {"role": "user", "content": "Extract name and age from: 'My name is Bob and I am 25 years old.'"},
        {"role": "assistant", "content": "{"}  # ← prefill: forces JSON from char 1
    ]
)
# response.content[0].text will be the rest of the JSON after "{"
raw = "{" + response.content[0].text
import json
data = json.loads(raw)
print(data)
Python — prefill to skip disclaimer preambles
# Skip "Certainly! I'd be happy to help..." preambles
response = client.messages.create(
    model="claude-opus-4-5",
    max_tokens=512,
    messages=[
        {"role": "user", "content": "List 5 Python best practices."},
        {"role": "assistant", "content": "Here are 5 Python best practices:\n1."}
    ]
)
# Claude continues from "1." directly
💡 Prefill + JSON

The most reliable way to get pure JSON: set system='Respond in JSON only', AND prefill with '{'. This double-locks the format.

6. Chain-of-Thought Prompting

Force Claude to reason step-by-step before answering. This dramatically improves accuracy on math, logic, and multi-step problems. Use XML tags to separate reasoning from the final answer.

Python — CoT with XML tags
def solve_with_cot(problem: str) -> dict:
    response = client.messages.create(
        model="claude-opus-4-5",
        max_tokens=1024,
    system="""Solve problems step-by-step. Use: <thinking>[reasoning]</thinking><answer>[answer]</answer>""",
        messages=[{"role": "user", "content": problem}]
    )
    text = response.content[0].text

    # Parse thinking and answer from XML
    import re
    thinking = re.search(r"<thinking>(.*?)</thinking>", text, re.DOTALL)
    answer = re.search(r"<answer>(.*?)</answer>", text, re.DOTALL)
    return {
        "thinking": thinking.group(1).strip() if thinking else "",
        "answer": answer.group(1).strip() if answer else text,
    }

result = solve_with_cot("A train leaves at 9am going 60mph. Another at 11am going 80mph from the same station. When does the second catch the first?")
print("Thinking:", result["thinking"])
print("Answer:", result["answer"])

7. Few-Shot Examples

Provide 2–5 example input/output pairs in the messages array to teach Claude exactly the format you want. Far more reliable than describing the format in words.

Python — few-shot sentiment classifier
examples = [
    ("I love this product!", "positive"),
    ("Terrible experience, avoid.", "negative"),
    ("It arrived on time.", "neutral"),
    ("Best purchase I ever made.", "positive"),
    ("The packaging was broken.", "negative"),
]

def build_few_shot_messages(text: str) -> list[dict]:
    messages = []
    for inp, out in examples:
        messages.append({"role": "user", "content": inp})
        messages.append({"role": "assistant", "content": out})
    messages.append({"role": "user", "content": text})
    return messages

response = client.messages.create(
    model="claude-haiku-4-5-20251001",
    max_tokens=10,
    system="Classify sentiment. Reply with only: positive, negative, or neutral.",
    messages=build_few_shot_messages("Shipping was delayed but product is fine.")
)
print(response.content[0].text)  # neutral

8. Key Parameters Reference

  • model (required) — model ID: claude-opus-4-5, claude-sonnet-4-6, claude-haiku-4-5-20251001
  • max_tokens (required) — hard ceiling on output tokens (1–8192+)
  • messages (required) — alternating user/assistant array
  • system (optional) — system prompt string
  • temperature (optional) — 0.0–1.0, default 1.0; lower = more deterministic
  • top_p (optional) — nucleus sampling threshold
  • top_k (optional) — limit token candidates (lower = more focused)
  • stop_sequences (optional) — list of strings to halt generation
  • stream (optional) — boolean, enable SSE streaming
  • metadata.user_id (optional) — for safety monitoring
Python — all parameters together
response = client.messages.create(
    model="claude-opus-4-5",
    max_tokens=512,
    temperature=0.3,          # focused, deterministic
    top_p=0.9,
    top_k=50,
    stop_sequences=["", "###"],
    system="You are a concise technical writer.",
    messages=[{"role": "user", "content": "Explain recursion."}],
    metadata={"user_id": "user_abc123"}
)

9. Production Patterns

Python — retry with exponential backoff
import time, random
from anthropic import Anthropic, RateLimitError, APIStatusError

def call_with_retry(client, max_retries=5, **kwargs):
    for attempt in range(max_retries):
        try:
            return client.messages.create(**kwargs)
        except RateLimitError:
            wait = (2 ** attempt) + random.random()
            print(f"Rate limited. Waiting {wait:.1f}s...")
            time.sleep(wait)
        except APIStatusError as e:
            if e.status_code >= 500:
                time.sleep(2 ** attempt)
            else:
                raise  # 4xx errors: don't retry
    raise Exception("Max retries exceeded")
Python — cost estimation before calling
COST_PER_MTK = {
    "claude-opus-4-5":        {"in": 15.0, "out": 75.0},
    "claude-sonnet-4-6":      {"in": 3.0,  "out": 15.0},
    "claude-haiku-4-5-20251001": {"in": 0.25, "out": 1.25},
}

def estimate_cost(model: str, input_tokens: int, output_tokens: int) -> float:
    rates = COST_PER_MTK.get(model, COST_PER_MTK["claude-haiku-4-5-20251001"])
    return (input_tokens * rates["in"] + output_tokens * rates["out"]) / 1_000_000

# After a call
cost = estimate_cost(
    "claude-opus-4-5",
    response.usage.input_tokens,
    response.usage.output_tokens
)
print(f"This call cost ~${cost:.6f}")
